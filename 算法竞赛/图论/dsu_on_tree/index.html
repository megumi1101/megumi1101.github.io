DSU on Tree

DSU on Tree 也叫树上启发式合并。它主要用来解决一类子树统计问题：

  • 求每个点子树内有多少种颜色；
  • 求每个点子树内出现次数最多的颜色和；
  • 求每个点子树内满足某些出现次数条件的值。

这类问题的共同点是：每个询问都和一棵子树有关，而且可以通过 add(x) 和 del(x) 把一个点加入 / 移出当前维护的数据结构。

核心思想

如果对每个点都暴力扫描一遍子树，复杂度最坏是 O(n2)O(n^2)O(n2)。

DSU on Tree 的做法是：

  1. 先求出每个点的重儿子，也就是子树最大的儿子。
  2. 处理一个点 uuu 时，先递归处理所有轻儿子，并且处理完后清空它们的贡献。
  3. 再递归处理重儿子，并且保留重儿子的贡献。
  4. 然后把所有轻儿子的整棵子树重新加入当前数据结构。
  5. 最后加入 uuu 自己，此时数据结构里恰好就是 uuu 的整棵子树。

为什么这样快？因为每个点作为轻子树的一部分被重新加入时，子树规模会不断变大。一个点沿着祖先往上，最多经历 O(log⁡n)O(\log n)O(logn) 次轻边，所以总复杂度通常是 O(nlog⁡n)O(n\log n)O(nlogn)。如果 add 和 del 是 O(1)O(1)O(1)，整题就是 O(nlog⁡n)O(n\log n)O(nlogn)。

需要的预处理

这份写法直接复用树链剖分模板里的数组：

  • ed[u]：只保留儿子的邻接表，并且 ed[u][0] 是重儿子；
  • in[u]：DFS 序；
  • out[u]：子树结束位置，子树区间是 [in[u],out[u])[in[u], out[u])[in[u],out[u])；
  • seq[i]：DFS 序为 iii 的节点编号。
