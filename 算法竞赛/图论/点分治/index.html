点分治

点分治用来处理树上的路径统计问题，比如：

  • 是否存在长度为 kkk 的路径；
  • 统计满足某种条件的点对；
  • 查询若干个距离是否能在树上出现。

核心思想：每次找当前连通块的重心，统计所有经过这个重心的路径，然后删掉重心，递归处理剩下的子树。

因为重心删掉后，每个子连通块大小都不超过当前大小的一半，所以递归层数是 O(log⁡n)O(\log n)O(logn)。

基本流程

  1. 找当前连通块的重心 root。
  2. 统计所有经过 root 的路径贡献。
  3. 标记 root 已删除。
  4. 对 root 的每棵未删除子树递归点分治。

模板里常用的数组：

  • siz[u]：当前连通块中，以 u 为根的子树大小；
  • vis[u]：u 是否已经作为重心被删掉；
  • root：当前连通块的重心；
  • mxt：删掉某点后最大连通块的大小，越小越好。

例题模型

下面这份模板解决的问题是：

给一棵带边权的树和 mmm 个询问 qiq_iqi​，判断树上是否存在一对点，距离恰好为 qiq_iqi​。

对当前重心 u，如果一条路径经过 u，那么：

dist(x,y)=dist(x,u)+dist(y,u)dist(x, y) = dist(x, u) + dist(y, u) dist(x,y)=dist(x,u)+dist(y,u)

所以可以用一个集合 pre 存已经处理过的子树里所有点到 u 的距离。枚举当前子树里的距离 it，检查 q[i] - it 是否在 pre 中即可。

注意：要一棵子树一棵子树地处理。当前子树先和之前的 pre 匹配，匹配完再加入 pre，这样可以避免把同一棵子树内部的路径误认为经过重心。

模板代码

这份模板里，每个距离还会枚举所有询问，所以复杂度是：

O(nmlog⁡n)O(nm\log n) O(nmlogn)
