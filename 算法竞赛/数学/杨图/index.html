杨图 (Young Tableaux)

杨图是组合数学与群表示论中的经典工具。在算法竞赛中，它常被用来把序列问题、括号/路径计数问题转化为形状计数问题。

常见关联：

  • 最长递增子序列、最长递减子序列。
  • 标准杨表计数。
  • 受限网格路径与卡特兰数。
  • 整数分拆。

1. 基本概念

1.1 整数分拆与杨图形状

一个整数 nnn 的分拆可以写成

n=a1+a2+⋯+ak,a1≥a2≥⋯≥ak≥1.n = a_1 + a_2 + \cdots + a_k,\qquad a_1 \ge a_2 \ge \cdots \ge a_k \ge 1. n=a1​+a2​+⋯+ak​,a1​≥a2​≥⋯≥ak​≥1.

它可以唯一对应一个杨图形状：一共有 kkk 行，第 iii 行有 aia_iai​ 个方格，所有行左对齐。

1.2 标准杨表

将整数 111 到 nnn 填入一个含有 nnn 个格子的杨图中，每个数字恰好出现一次。如果满足：

  • 每一行从左到右严格递增。
  • 每一列从上到下严格递增。

则称为标准杨表 (Standard Young Tableau, SYT)。

2. 钩长公式

当需要计算某个固定形状的标准杨表数量时，可以使用钩长公式。

2.1 钩长定义

对于杨图中的任意格子 (i,j)(i,j)(i,j)，其钩长 hi,jh_{i,j}hi,j​ 定义为：

hi,j=右侧格子数+下方格子数+1.h_{i,j} = \text{右侧格子数} + \text{下方格子数} + 1. hi,j​=右侧格子数+下方格子数+1.

这里的 111 表示格子自身。

2.2 公式

设杨图形状为 λ\lambdaλ，总格子数为 nnn。该形状对应的标准杨表数量为

fλ=n!∏(i,j)∈λhi,j.f^\lambda = \frac{n!}{\prod_{(i,j)\in \lambda} h_{i,j}}. fλ=∏(i,j)∈λ​hi,j​n!​.

其中乘积遍历杨图中的所有格子。

3. RS 插入算法与 Schensted 定理

RS (Robinson-Schensted) 算法可以将一个排列双射到一对同形状的标准杨表。竞赛中最常用的是其中的插入表，因为它的形状可以直接读出原序列的最长单调子序列信息。

可以先把它理解成一种“维护若干递增行”的算法：新数先尝试放入第一行，如果会破坏递增性，就把这一行里第一个比它大的数挤下去；被挤下去的数再用同样方式插入下一行。

3.1 插入与挤出规则

将序列中的元素 xxx 依次插入杨表：

  1. 在当前行找到第一个严格大于 xxx 的元素 yyy。
  2. 用 xxx 替换 yyy。
  3. 被替换出的 yyy 进入下一行，继续执行同样的过程。
  4. 如果当前行没有大于 xxx 的元素，则把 xxx 追加到当前行末尾，插入结束。

这个过程也常被称为 bumping。

注意，普通 RS 对排列最自然，因为所有数互不相同。若序列中允许相等元素，使用 upper_bound 还是 lower_bound 会影响它对应的是严格递增还是非严格递增版本，写题时要和题目中的 LIS 定义保持一致。

3.2 插入例子

以序列

4, 1, 3, 2, 6, 5, 74,\ 1,\ 3,\ 2,\ 6,\ 5,\ 7 4, 1, 3, 2, 6, 5, 7

为例，从空表开始逐个插入。

插入 444：第一行为空，直接放入。

插入 111：第一行中第一个大于 111 的数是 444，于是 111 替换 444，被挤出的 444 进入第二行。

插入 333：第一行没有大于 333 的数，直接追加到第一行末尾。

插入 222：第一行中第一个大于 222 的数是 333，用 222 替换它；被挤出的 333 插入第二行。第二行当前为 4，第一个大于 333 的数是 444，于是 333 替换 444，被挤出的 444 继续进入第三行。

插入 666：第一行没有大于 666 的数，追加到第一行末尾。

插入 555：第一行中第一个大于 555 的数是 666，用 555 替换它；被挤出的 666 插入第二行。第二行当前为 3，没有大于 666 的数，所以 666 追加到第二行末尾。

插入 777：第一行没有大于 777 的数，追加到第一行末尾。

最终形状是 (4,2,1)(4,2,1)(4,2,1)。第一行长度为 444，对应原序列的 LIS 长度，例如 1,2,5,71,2,5,71,2,5,7。第一列长度为 333，对应原序列的 LDS 长度，例如 4,3,24,3,24,3,2。

这个例子里最关键的是插入 222 的那一步：它先把 333 从第一行挤到第二行，又把 444 从第二行挤到第三行。一个数被挤出后并没有消失，而是继续向下一行寻找位置；正是这种“逐层下沉”的过程，让最后的图形同时记录了递增和递减结构。

3.3 Schensted 定理

将序列转化为杨表后，杨表形状直接反映最长子序列信息：

  • 第一行长度等于原序列的 LIS 长度。
  • 第一列长度等于原序列的 LDS 长度。

这里的 LIS 指最长递增子序列，LDS 指最长递减子序列。

更一般地，如果最终杨表形状为

λ=(λ1,λ2,…),\lambda=(\lambda_1,\lambda_2,\ldots), λ=(λ1​,λ2​,…),

那么前 kkk 行的总长度 λ1+⋯+λk\lambda_1+\cdots+\lambda_kλ1​+⋯+λk​，可以理解为从序列中选出若干元素，并把它们分成 kkk 条递增子序列时，最多能选出的元素数。这个结论比只看第一行更强，但竞赛中最常见的用法仍然是读出 LIS 和 LDS。

4. 几何模型映射

杨表中“列严格递增”的限制天然适合描述不越界的路径约束。核心思想是：把路径的每一步按时间编号，然后根据这一步的方向，把编号填入杨表的不同行。

因为时间编号是从小到大填入的，所以每一行天然递增；真正产生限制的是列递增。列递增会要求某一类步数的第 jjj 次出现，必须早于另一类步数的第 jjj 次出现，这正好对应“走路过程中不能越界”。

4.1 二维寻路与卡特兰数

典型题型：在网格中只能向右或向上走，并要求路径不越过对角线。例如始终满足

向右步数≥向上步数.\text{向右步数} \ge \text{向上步数}. 向右步数≥向上步数.

假设一共走 nnn 次向右和 nnn 次向上。把 2n2n2n 个步骤按发生时间编号为 1,2,…,2n1,2,\ldots,2n1,2,…,2n：

  • 若第 ttt 步向右，就把 ttt 填入第一行的下一个空格。
  • 若第 ttt 步向上，就把 ttt 填入第二行的下一个空格。

这样会得到一个两行、每行 nnn 个格子的表。由于编号按时间递增，每一行一定从左到右递增。

列递增的含义是：第一行第 jjj 个数小于第二行第 jjj 个数。翻译回路径，就是第 jjj 次向右必须早于第 jjj 次向上。也就是说，在任何时刻，向右步数都不少于向上步数，路径不会越过对角线。

反过来，任意一个 2×n2\times n2×n 矩形标准杨表，也能唯一还原成一条合法路径：编号在哪一行，就表示对应时刻走哪个方向。因此这两类对象一一对应。

所以这类问题也等价于合法括号序列计数。它可以映射为一个 2×n2\times n2×n 的矩形杨图，因此方案数为第 nnn 个卡特兰数：

Cn=1n+1(2nn).C_n = \frac{1}{n+1}\binom{2n}{n}. Cn​=n+11​(n2n​).

也可以用钩长公式直接验证。对于形状 (n,n)(n,n)(n,n)，钩长乘积为

(n+1)n(n−1)⋯2⋅n(n−1)⋯1,(n+1)n(n-1)\cdots 2 \cdot n(n-1)\cdots 1, (n+1)n(n−1)⋯2⋅n(n−1)⋯1,

因此

f(n,n)=(2n)!(n+1)! n!=1n+1(2nn).f^{(n,n)} = \frac{(2n)!}{(n+1)!\,n!} = \frac{1}{n+1}\binom{2n}{n}. f(n,n)=(n+1)!n!(2n)!​=n+11​(n2n​).

4.2 更多维情况（三维为例）

三维情形只是把二维的两行表推广为三行表。若三种方向各走 nnn 次，并且路径过程中始终满足

X≥Y≥Z,X \ge Y \ge Z, X≥Y≥Z,

则把所有步骤按时间编号后：

  • 走 XXX 方向时，把编号填入第一行。
  • 走 YYY 方向时，把编号填入第二行。
  • 走 ZZZ 方向时，把编号填入第三行。

列递增会同时保证“第 jjj 次走 XXX 早于第 jjj 次走 YYY”和“第 jjj 次走 YYY 早于第 jjj 次走 ZZZ”，也就是全过程中 X≥Y≥ZX\ge Y\ge ZX≥Y≥Z。
